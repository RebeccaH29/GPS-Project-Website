Assigning your own data to GPSCs


After installing popPUNK as per instructions at https://poppunk.readthedocs.io/en/latest/installation.html and downloading the GPS reference database “GPS_query.tar.bz2” Database and the GPSC designations for the references “gpsc_definitive.csv” from CSV https://www.pneumogen.net/gps/assigningGPSCs


Uncompress the GPS_query.tar.bz2 database file.


    Files required to run GPSC assignment using popPUNK:
  1. queries.txt (a list of paths to assemblies you wish to assign)
  2. GPS_query (GPS reference database, GPS_query.tar.bz2, uncompressed)
  3. gpsc_definitive.csv (Published GPSC designations for the references)

output directory name is assigned using --output

number of threads can be changed using --threads


Run GPSC assignment:

poppunk --assign-query --ref-db GPS_query --distances GPS_query/GPS_query.dists --model-dir GPS_query --q-files queries.txt --output GPSC_assignment --threads 8 --no-stream --full-db --external-clustering gpsc_definitive.csv


Outputs:

_clusters.csv (popPUNK clusters arbitrary nomenclature)
